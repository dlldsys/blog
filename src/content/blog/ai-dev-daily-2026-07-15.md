## 涨星最快项目

### 1. Zackriya-Solutions/meetily — 隐私优先的本地 AI 会议助手

Meetily 本周以 23,583 星、周增 7,440 星登顶 GitHub Trending。这是一个 Rust/Tauri 桌面应用，使用本地 Whisper/Parakeet 转录和 Ollama 摘要，支持 Metal、CoreML、CUDA、Vulkan 硬件加速，让团队无需将会议音频发送到云端即可获得 AI 会议笔记。

GitHub：[Zackriya-Solutions/meetily](https://github.com/Zackriya-Solutions/meetily) · 来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

### 2. asgeirtj/system_prompts_leaks — AI 系统提示词泄露库

收录了 Anthropic、OpenAI、Google、Microsoft、Meta、xAI、Cursor、Perplexity 等 10+ 厂商的提取系统提示词，本周达 56,733 星、周增 7,155 星。7 月 10 日新增了 GPT-5.6 Sol 和 Codex GPT-5.6 的提示词材料，包括 Claude Fable 5 长达 3,800 行、19 万 Token 预算的提示词。

GitHub：[asgeirtj/system_prompts_leaks](https://github.com/asgeirtj/system_prompts_leaks) · 来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

### 3. stablyai/orca — 并行 Coding Agent 舰队管理环境

Orca 本周达 16,991 星、周增 4,481 星。基于 Electron + React 19 + TypeScript 构建，可在独立 git worktree 中并行运行 Claude Code、Codex、Grok、Cursor、GitHub Copilot 等多个 CLI Agent 舰队，支持桌面与移动端。v1.4.137 稳定版于 7 月 12 日发布。

GitHub：[stablyai/orca](https://github.com/stablyai/orca) · 来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

### 4. diegosouzapw/OmniRoute — 自托管多模型路由网关

OmniRoute 本周达 16,228 星、周增 4,506 星。作为自托管的 OpenAI 兼容网关，v3.8.46 新增 pipeline 顺序执行策略、Gemini /v1beta 工具调用、代理池 IP 轮换等功能，正从单纯的模型聚合向运维层演进。

GitHub：[diegosouzapw/OmniRoute](https://github.com/diegosouzapw/OmniRoute) · 来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

### 5. openai/codex-plugin-cc — Codex 运行在 Claude Code 内的官方插件

OpenAI 官方发布的插件，让 Codex 直接在 Claude Code 中运行，暴露 `/codex` 命令实现代码审查、对抗性审查、任务救援、会话迁移等功能。本周达 28,059 星、周增 2,803 星，代表了"Claude 写实现 + Codex 做审查"的分工协作新范式。

GitHub：[openai/codex-plugin-cc](https://github.com/openai/codex-plugin-cc) · 来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

## 新工具 & CLI

### 1. OpenAI openai-cli — 官方命令行工具

OpenAI 发布官方 CLI 工具 openai-cli，采用 Apache 2.0 开源协议，用 Go 语言实现。开发者无需编写 SDK 代码即可在终端中直接调用 OpenAI API 和 Responses API，支持资源化命令结构 `openai [resource] <command> [flags]`，适合脚本自动化和开发调试。

GitHub：[openai/openai-cli](https://github.com/openai/openai-cli) · 来源：[AI-Damn](https://ai-damn.com/openai-s-new-cli-tool-brings-ai-power-to-your-terminal-1778213693127) · [CSDN](https://blog.csdn.net/qq_41185868/article/details/160958827)

### 2. GitHub Copilot CLI with HyDRA — 智能模型路由

GitHub 更新 Copilot CLI，引入名为 HyDRA 的内部路由系统，可自动分析任务复杂度、推理深度和工具编排需求，将任务路由到最合适的 AI 模型。这一自动化策略旨在优化成本与性能平衡。

来源：[4sysops](https://4sysops.com/archives/github-copilot-cli-introduces-intelligent-task-routing-with-hydra/)

### 3. Google Genkit Agents API — 统一 Agent 开发接口

Google 发布 Genkit Agents API 预览版（TypeScript/Go），将消息历史、工具循环、流式传输和状态持久化封装在单一 `chat()` 接口后。支持"分离回合"让 Agent 在客户端断开后继续工作，以及可中断工具提供人工审查能力。

来源：[InfoQ](https://www.infoq.com/ai-ml-data-eng/news/1/)

### 4. Augment Auggie CLI — 终端 AI 编程代理

Augment 发布 Auggie CLI，作为代理型 CLI 工具，拥有理解整个代码仓库的上下文引擎。原生集成 VS Code，支持终端内 AI 辅助编码工作流，定位为企业级 AI 开发平台。

来源：[AI-Damn](https://ai-damn.com/augment-s-auggie-cli-aims-to-transform-developer-workflows-1754263237246)

### 5. TencentCloud/CubeSandbox — AI Agent MicroVM 沙箱

腾讯云开源的 AI Agent 代码执行沙箱，使用 RustVMM 和 KVM MicroVM，冷启动低于 60ms，单实例内存开销低于 5MB，一台 96 核裸金属主机可运行 2000+ 并发沙箱。v0.5.0 新增 AutoPause/AutoResume 和 ARM64 原生支持。

GitHub：[TencentCloud/CubeSandbox](https://github.com/TencentCloud/CubeSandbox) · 来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

## 编程方式

### 1. OpenAI 工程师用 Agentic Loops 自动化 40% 编程工作流

OpenAI 工程团队正从手动提示转向 Agentic Loops（代理循环）方法论，成功自动化了 40% 的编程工作流。通过实施自主执行循环，团队将人类意图与语法级实现解耦，工程师角色转向高层架构和循环设计。这标志着行业从"LLMOps 仅看模型大小"转向"复杂工程实践驱动效率"。

来源：[TheNextGenTechInsider](https://www.thenextgentechinsider.com/pulse/openai-engineers-use-agentic-loops-to-automate-40-of-coding-workflows)

### 2. 2026 年 AI 编程从"代码补全"进入"工程生成"时代

2026 年被定义为"工程生成元年"：AI 编程工具不再是"帮你写代码的助手"，而是"帮你交付工程的合作方"。从需求理解、接口设计、表结构设计到完整工程代码，实现一站式交付。Agent 可自主生成子 Agent，进行递归规划。

来源：[搜狐](https://m.sohu.com/a/1049984822_120959233/) · [CSDN](https://blog.csdn.net/yanceyxin/article/details/162586505)

### 3. 多 Agent 并行编排成为新范式

codex-plugin-cc（Codex 在 Claude Code 内运行）和 Orca（并行 Agent 舰队管理）代表了"多 Agent 分工协作"的新范式：一个 Agent 写实现，另一个 Agent 做审查，第三个 Agent 测试。开发者退后一步成为系统架构师和监督者，而非逐行编写代码。

来源：[NeoDrop](https://neodrop.ai/post/tjSLeZwktnr)

Satya Nadella 指出，提示词、纠正和 Agent 行为日志正在成为企业的运营知识产权。企业需建立编排层实现多模型路由——高端模型处理复杂推理，低成本模型处理日常任务，本地模型处理敏感信息。模型可移植性不再是可选项，而是战略必需品。

来源：[Hipther](https://hipther.com/latest-news/2026/07/14/115093/ai-dispatch-microsoft-satya-nadella-meta-hyperion-acuitymd-simform-truemorph-and-the-ai-workforce-july-14-2026/)
